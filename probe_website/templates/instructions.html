Instructions

This page contains instructions on how to set up probes.

  1. Go to the Download image tab and download the image for either Raspberry Pi 1 or Raspberry Pi 2/3 (RPi 2 and 3 uses the same image)
  2. Install the image on an SD card with at least 8 GB of space.
    • On Linux: First run lsblk to find the name of the SD card (may be called e.g. sde or mmcblk0), and then run the following command: dd if=/path/to/image of=/dev/mmcblk0 bs=1M conv=sync (may need to be run as root), where mmcblk0 is replaced with the name of your SD card.
    • On Mac: First run diskutil list to find the name of the SD card, and then run the following command: dd if=/path/to/image of=/dev/rdiskX bs=1m (may need to be run as root, sudo -s), where /dev/rdiskX is your SD card.
    • On Windows: Use Win32DiskImager (download) to copy the image onto the SD card. Choose the image file from your drive, select the SD card as Device and click Write.
  3. Go to the Databases tab, choose which database types you want to use and add required information. If you only want to use UNINETT's databases, you can skip this step.
  4. Now you must register the RPi under the Probes tab. The WiFi dongle's MAC address will be printed on the dongle itself. Note that this registration can be done at any time, but the RPi will not associate with the server before it is done.
    1. Click 'Edit' on a probe and add the network credentials of the network you want the probe to measure. You can also change the interval at which each WiFi test should be run. Note that if you're adding multiple probes, and they will all have the same (or almost the same) credentials, make sure to click 'Save configuration as default', as this will make sure all probes added after that will have these credentials.
    2. Note that saving as default will not affect already added probes.
  5. Insert the SD card into the Raspberry Pi and connect the following to it:
    • Ethernet cable
    • WiFi dongle
    • Power cable
  6. The RPi will automatically initiate a setup procedure at first boot, where it installs necessary programs and associates itself with the server. Do not cut the power while the RPi is doing this. Note that this can take 20-30 minutes.
  7. The RPi is ready when you see a green 'Associated' button and 'Connected' text under 'Probe status' in the Probes tab (you may need to refresh the page). If the button is red, click on it to renew the association period.
    1. If the RPi is not ready after 30-60 min, make sure the ethernet cable has a working internet connection.
  8. If the network credentials have already been added, you can now click on 'Push configuration to probes' to update the probes with the added information. This can take a while. Note that you should wait for all your probes to be associated before doing this, as all associated probes will be updated at once.
  9. After the update is done ('Update status' says completed), power off the RPi by disconnecting/pulling out the WiFi dongle. This will initiate a shutdown procedure, and is how the probe must be shut down to avoid file corruption. After the green light on the RPi stops blinking (will take a few seconds), you can cut the power and remove the ethernet cable. Don't forget to plug the WiFi dongle back in.
  10. The probe will automatically start measuring the network next time power is connected, as long as no ethernet cable is connected. This is to avoid doing measurements over cable instead of WiFi.
